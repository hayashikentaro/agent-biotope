# Harvest Model

Agent Biotope is not a wilderness.

It is a cultivated artificial ecosystem. It should produce value for an external gardener, organization, product, or world.

The question is not only how agents live inside the habitat. The question is how value is harvested without depleting the habitat.

## Core Cycle

```text
Nutrients in
  -> metabolism
  -> harvest out
  -> residue
  -> decomposition
  -> return to substrate
  -> healthier next cycle
```

Harvest is not complete until some value returns to the habitat.

## Nutrients

Nutrients are the inputs that feed the habitat.

Examples:

- goals;
- issues;
- constraints;
- repo state;
- failing tests;
- logs;
- user feedback;
- product signals;
- human taste;
- examples of good output;
- architecture doctrine;
- compute budget;
- time and attention.

Computation alone is not enough. Nutrients give computation direction and selectivity.

## Metabolism

Metabolism is the internal processing that turns nutrients into outputs and residue.

Examples:

- worker execution;
- review;
- testing;
- summarization;
- decomposition;
- watchdog enforcement;
- integration;
- human decision.

Healthy metabolism does not merely produce artifacts. It separates harvestable value from residue and returns useful residue to the substrate.

## Harvest

Harvestable outputs are value that can be taken out of the habitat.

Examples:

- merged code;
- accepted documentation;
- reviewed pull requests;
- closed issues;
- validated design decisions;
- risk reports;
- reusable prompts;
- test cases;
- refined patterns;
- product insights;
- operational procedures.

A harvest should be useful outside the habitat.

## Residue

Residue is what remains after attempted work.

Examples:

- failed branches;
- rejected diffs;
- stale tasks;
- noisy logs;
- ambiguous requirements;
- flaky tests;
- reviewer disagreements;
- duplicated concepts;
- incomplete plans;
- abandoned experiments.

Residue is not always waste. Often it is unprocessed nutrient.

## Decomposition And Return

A decomposer turns residue into habitat improvement.

Examples:

- failed attempt summary;
- new watchdog rule;
- stronger reviewer profile;
- better task decomposition pattern;
- clearer prompt template;
- additional test case;
- glossary update;
- architecture note;
- tighter capability boundary;
- cleanup of abandoned branches.

The return loop is what distinguishes regenerative harvest from extraction.

## Overharvesting

Overharvesting happens when outputs are extracted but the substrate is not replenished.

Signs include:

- many merged changes but worsening architecture coherence;
- growing stale branch count;
- repeated failures without rulebook updates;
- tests and docs lagging behind generated code;
- reviewers producing reports but profiles not improving;
- human attention backlog increasing;
- watchdogs alerting frequently without systemic fixes;
- more agents added while decomposition capacity stays flat.

Overharvesting may look productive in the short term. It makes the habitat brittle over time.

## Regenerative Harvest

A regenerative harvest improves both the outside world and the habitat.

Examples:

```text
merged feature
  + tests added
  + architecture rule clarified
  + reviewer profile improved

failed worker branch
  + failure summarized
  + task split pattern created
  + watchdog check added

rejected design direction
  + decision recorded
  + glossary clarified
  + future prompts constrained
```

## Design Rule

```text
Harvest should feed the next substrate.
```

The gardener may take value out of the habitat, but some value must return as structure, knowledge, tests, boundaries, or decomposition capacity.

A cultivated agent ecosystem should become easier to operate as it produces value.
